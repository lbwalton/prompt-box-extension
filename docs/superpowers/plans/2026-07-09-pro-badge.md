# Pro Badge (verified-style seal) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Show a teal rosette seal (verified-checkmark style) in the popup header and Account panel: vivid for Pro users, dimmed for everyone else, painting instantly from a `pb_is_pro` cache that refreshes from the existing entitlement fetch.

**Architecture:** Static inline SVG badges in `popup.html` (header `<button>` + Account panel `<span>`), toggled by one `renderProBadge()` function in `popup.js` that reads the `pb_is_pro` boolean from `chrome.storage.local`. The cache is written only on DEFINITIVE entitlement answers (inside the existing `refreshCloudOption()`) and set false on sign-out. Clicking the header badge opens the Settings panel's Account tab through the existing `showTagManagement()` + `.settings-tab` click mechanics.

**Tech Stack:** Vanilla JS MV3 popup (no build step), inline SVG, existing CSS custom properties. Lint via `npm run lint`, security via `npm run security`.

**Spec:** `docs/superpowers/specs/2026-07-09-pro-badge-design.md`
**Branch:** work on `feature/pro-b2-phase2-cloud-sync` (the badge rides the Phase 2 branch and ships in the same future 4.0.0 release; no version bump now).

## Global Constraints

- No build step; CSP stays `script-src 'self'; object-src 'self'`. All markup is static inline SVG in popup.html; no `innerHTML` anywhere in this feature; new rendered text is static literals.
- Colors come from existing tokens only: seal fill `var(--color-info)` (#0d9488 light / #14b8a6 dark) via `currentColor`; check stroke `var(--color-surface, #fff)`. No new color values.
- Dimmed (non-Pro) state is CSS only: `filter: grayscale(1); opacity: 0.45;` toggled by a `dimmed` class. Same markup for both states.
- `pb_is_pro` (chrome.storage.local, boolean, absent = false) is DISPLAY STATE ONLY. It gates nothing: cloud-mode gating still runs on the live entitlement in `refreshCloudOption`, unchanged.
- Cache write rules (must match the cloud-mode demotion rule exactly): write `pb_is_pro` ONLY when `fetchEntitlement()` returns non-null (definitive), or on sign-out (set false). A null/transient result never changes the cache and never dims the badge.
- Files touched: `popup.html`, `popup.js`, `supabase/TEST-phase2-cloud-sync.md` only. Do NOT touch sync-engine.js, sync-auth.js, background.js, content.js, manifest.json, or the DB.
- Free-user zero-regression: for signed-out/free users the ONLY visible change is the two dimmed badge elements; every existing flow behaves identically.
- Before every commit: `npm run lint` and `npm run security` pass with 0 errors.
- No automated test runner. Verification per task = `node --check` + lint/security + the manual checks listed in the task (unpacked-extension reload; deeper live checks are batched into the runbook).

**Manual reload procedure:** `chrome://extensions` → Prompt Box → reload, then reopen the popup.

- [ ] **Step 1: Add `renderProBadge` and `openAccountPanel`**

In `popup.js`, directly above the `// ---- Account (Prompt Box Pro) ----` comment (~line 223), add:

```js
// ---- Pro badge (verified-style seal) ----
// Paints instantly from the pb_is_pro cache; refreshCloudOption updates the
// cache on every DEFINITIVE entitlement answer. Display state only — gates nothing.
function renderProBadge() {
  chrome.storage.local.get(['pb_is_pro'], function (r) {
    const isPro = r.pb_is_pro === true;
    const label = isPro ? 'Prompt Box Pro member' : 'Get Prompt Box Pro';
    const header = document.getElementById('proBadgeHeader');
    const account = document.getElementById('proBadgeAccount');
    const word = document.getElementById('proBadgeWord');
    if (header) {
      header.classList.toggle('dimmed', !isPro);
      header.title = label;
      header.setAttribute('aria-label', label);
    }
    if (account) account.classList.toggle('dimmed', !isPro);
    if (word) word.style.display = isPro ? 'inline' : 'none';
  });
}

// Open the Settings panel on the Account tab (used by the badge click).
function openAccountPanel() {
  const tagManagement = document.getElementById('tagManagement');
  if (tagManagement && tagManagement.style.display !== 'block') {
    showTagManagement();
  }
  const accountTab = document.querySelector('.settings-tab[data-tab="account"]');
  if (accountTab) accountTab.click();
  if (tagManagement) tagManagement.scrollIntoView({ behavior: 'smooth', block: 'start' });
}
```

(`showTagManagement()` toggles, so it is called only when the panel is hidden. `accountTab.click()` reuses the existing tab-switching listener; no new switching logic.)

In the sign-out handler inside `setupAccountUI` (popup.js ~line 257-263), after the `resetSyncState` line, add the cache write so the handler reads:

```js
  if (signOutBtn) {
    signOutBtn.addEventListener('click', async function () {
      await PBAuth.signOut();
      try { await PBSync.resetSyncState(); } catch (e) { /* best effort */ }
      chrome.storage.local.set({ pb_is_pro: false }, renderProBadge);
      await renderAccount();
    });
  }
```

(Necessary because after sign-out `fetchEntitlement()` returns null, which by design never touches the cache; without this line a Pro badge would survive sign-out.)
